## 1. 基本思想

Kinakoは、既存言語が抱える「安全性」「複雑性」「学習コスト」の問題を軽減することを目的として設計されたプログラミング言語である。

設計において、次の原則を重視する。

### 1.1 単一責務

一つの概念は一つの責務のみを持つ。

型・権利・実装戦略・実装契約・分類などの異なる概念を分離し、それぞれが独立した責務を持つよう設計する。

### 1.2 最小概念

新しい概念は必要最小限に留める。

既存の概念の組み合わせで表現できるものは、新たな言語機能として導入しない。

### 1.3 明示的な責務

値の構造、アクセス権、実装戦略など、それぞれ異なる役割を明確に分離する。

暗黙に複数の意味を持つ構造は避け、それぞれ独立した概念として扱う。

### 1.4 単純な規則

利用者が理解しやすく説明しやすい規則を優先する。

複雑な特例や例外的な仕様よりも、少数の一貫した規則によって言語全体を構成する。

### 1.5 安全性

危険な操作は可能な限りコンパイル時に検出する。

また、安全性を実現するために複雑な概念を利用者へ要求するのではなく、単純な規則の組み合わせによって実現する。

### 1.6 実用性

安全性だけでなく、実際の開発における記述量・可読性・保守性も重視する。

頻繁に発生する煩雑な記述は、言語仕様によって自然に解消することを目指す。
