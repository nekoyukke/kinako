# Kinakoへの貢献 🌟

Kinakoへの貢献にご興味をお持ちいただきありがとうございます！このドキュメントでは、プロジェクトに参加するためのガイドラインと手順を説明します。

## 目次

- [行動規範](#行動規範)
- [貢献の方法](#貢献方法)

## 行動規範 📜

すべてのやり取りにおいて、敬意と建設的な姿勢を心がけてください。私たちは、すべての貢献者にとって歓迎され、誰もが参加できる環境を提供することに尽力しています。\
あなたのしたことでレンズを曇らせていないかを確認してください。

## 貢献方法 ❓

### 1. バグの報告 🐛

バグを発見しましたか？以下の情報を添えて `Issue` を作成してください。

- pythonバージョン
- kinakoバージョン
- ライブラリーバージョン
- コード
- エラーメッセージ

誰もがわかりやすいように題名は簡潔にしてください。簡潔ではないめんどくさい題名は歓迎されない可能性があります。\
また、 `kinako` 製ライブラリを使用している場合はそのライブラリのバグである可能性が高いです。純粋な `kinako` で発生するか検証して下さい。


### 2. ドキュメントの改善 📄

ドキュメントの改善は大歓迎です！\
以下の情報を添えて `Issue` を作成してください

- 不明瞭/正しくない場所
- その部分の引用
- 解決策/変更すべき表現

題名はドキュメント名を必ず含めてください。

### 3. ユーザとして使う 🖋

ユーザとして使ってもらうだけでもコミュニティ（1人）の助けになります！(喜ぶのでじゃんじゃん宣伝してください)\
使っていく過程でバグを見つけたら報告しましょう！\
できれば `Github` で `Issue` を作成するのがいいですが、開発者にDMでも送ってもいいです！
